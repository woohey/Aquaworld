# Aquaworld 沉浸式世界画卷重设计文档

日期 / Date: 2026-06-22

## 已确认方向 / Confirmed Direction

本阶段将 Aqua Biotope Atlas 从三栏式图谱应用重设计为 **沉浸式世界平铺画卷**。

用户确认的关键要求：

- 整体画面是一幅世界的平铺画卷。
- 画卷带流光溢彩的水纹效果，并且循环流动。
- 七大流域放在世界画卷中相对合理的位置。
- 附加信息以弹出信息框呈现，不再常驻首屏。
- 参考 `https://knight-l.github.io/sc-datav/#/demo2` 的部分流光、大屏层级和地图聚焦效果，但不转成纯数据大屏。
- 已确认采用方案 1：**整屏世界画卷单画布**。

## 产品定位 / Product Position

本次重设计的目标不是扩展功能，而是重塑第一印象。

打开应用后，用户首先看到的是一幅可以探索的水域世界图：深水绢本背景、抽象世界陆海轮廓、七个流域墨彩团块、轻微游动的鱼影和循环流动的光纹。

首屏不再像 dashboard，也不保留三栏常驻面板。地图是第一界面，信息只在用户点击或聚焦时浮出。

## 范围 / Scope

包含：

- 将主布局从三栏结构改为全屏沉浸式画卷。
- 新增 `WorldScrollCanvas` 作为核心视觉与交互承载组件。
- 七大流域按世界相对位置重新布局。
- 将流域详情、鱼种详情、筛选和混养提示迁移到弹窗或临时抽屉。
- 实现循环流光、水纹、鱼影浮动和选中态高亮。
- 保留现有静态数据、筛选逻辑、搜索逻辑和鱼种选择逻辑。
- 保证桌面首屏完整可看，移动端可浏览和点击。

不包含：

- 后端、CMS、账号、收藏、社交分享。
- 新增鱼种数据。
- 真实 GIS 地图或精确地理投影。
- WebGL / Three.js。
- 海报导出。
- 复杂粒子系统。

## 信息架构 / Information Architecture

### 首屏

首屏只保留四类常驻元素：

- 全屏世界画卷。
- 极轻标题层：`Aqua Biotope Atlas` 和 `观赏鱼原生地探索图谱`。
- 七大流域可点击水域团块。
- 少量鱼影标记，围绕对应流域分布。

首屏不常驻左侧筛选栏、右侧详情栏、底部混养栏和鱼种列表。

### 流域弹窗

点击流域后，在对应流域附近打开弹出信息框。

内容：

- 流域中文名和英文名。
- 一句 subtitle。
- 水体类型。
- 温度、pH、硬度范围。
- 代表鱼种小鱼签。
- 一个入口：查看该流域鱼群。

弹窗应像画卷题签或半透明数据牌，而不是厚重卡片。

### 鱼种弹窗

点击鱼影或流域弹窗里的鱼签后，弹出鱼种信息框。

内容：

- 中文名、英文名、学名。
- 原生流域。
- 水质参数。
- 性格、体长、泳层、饲养难度。
- 视觉标签与色彩标签。
- 简短故事备注。
- 混养关系摘要。

鱼种详情不再依赖右侧常驻面板。

### 临时控制层

搜索、筛选和重置保留，但不占据首屏。

入口：

- 一个右上角或左下角的轻量图标按钮，命名为“调境”。
- 点击后打开浮层抽屉。

抽屉内容复用现有筛选条件：

- 原生地。
- 水温。
- pH。
- 性格。
- 最大体长。
- 色彩。
- 饲养难度。
- 重置筛选。

搜索放在调境抽屉顶部，首屏标题层不常驻搜索输入框，以保持完全沉浸。

## 世界画卷布局 / World Scroll Layout

七大流域位置使用相对地理方位，不追求地图精确度：

- 亚马逊黑水：南美北部偏左中。
- 中美洲河流：左侧中下，靠近亚马逊上方或右上过渡位置。
- 非洲裂谷湖：中央偏下。
- 南亚稻田与缓流水域：中右偏下。
- 东南亚溪流：中右。
- 珊瑚礁浅滩：右上到右中海域，可作为更亮的高饱和浅滩。
- 中国南方溪流与稻田水域：右侧偏中上，靠近东亚位置。

画卷不是普通地图底图，而是抽象水域地貌：

- 大陆轮廓用低对比墨色团块暗示。
- 河流和洋流用细线、半透明路径和光纹暗示。
- 每个流域用自己的墨彩团块与微地貌纹理表达身份。
- 鱼影与流域绑定，选中流域时该区域鱼影更活跃。

## 视觉语言 / Visual Language

### 画面基调

- 背景：深水青黑、墨蓝、少量绢本纸纹。
- 流光：青绿、贝壳白、淡金、浅蓝，循环横向或斜向流动。
- 流域团块：低饱和墨彩，局部发光，不使用强霓虹。
- 珊瑚礁可略高饱和，但面积要受控。

### 参考效果吸收

从 `sc-datav` 借鉴：

- 地图中心化的第一视觉。
- 流动光线和扫描感。
- 点位聚焦后的信息浮层。
- 高对比暗场里的边线和发光层级。

不借鉴：

- 复杂数据大屏边框堆叠。
- 大量统计面板。
- 强科技蓝色单一主题。
- 纯仪表盘信息密度。

### 弹窗风格

弹窗采用“题签 + 水域数据牌”：

- 8 到 12px 圆角。
- 细淡金或水蓝边线。
- 半透明深水背景。
- 轻微背后模糊或暗化，但不遮住画卷。
- 标题清晰，参数紧凑。

## 动效设计 / Motion Design

### 循环流光

画卷底层需要持续循环流动：

- 使用 CSS 伪元素、SVG path stroke-dashoffset、background-position 或 transform 实现。
- 速度慢，避免数据大屏式急促闪烁。
- 流动方向以横向和轻微斜向为主，形成“水流过世界”的感觉。
- 支持 `prefers-reduced-motion`，减少或停止循环动画。

### 流域状态

流域有四种状态：

- 默认：低亮墨彩团块。
- Hover / focus：水纹局部增强，名称浮出。
- Selected：团块边缘淡金或青绿高亮，附近鱼影可见性增强。
- Filtered empty：降低透明度，不完全消失，保留世界结构。

### 鱼影状态

鱼影不做普通地图 marker。

- 默认是小型剪影或鳞片状光点。
- 微小漂移或摆尾效果。
- Hover / focus 显示鱼名。
- Selected 打开鱼种弹窗。

## 组件架构 / Component Architecture

### `App`

继续负责全局状态：

- `selectedHabitatId`
- `selectedFishId`
- `filters`
- `searchQuery`
- `activeOverlay`

`activeOverlay` 用于描述当前弹窗类型：

- `null`
- `{ type: 'habitat'; habitatId: string }`
- `{ type: 'fish'; fishId: string }`
- `{ type: 'tuning' }`

### `WorldScrollCanvas`

替代现有三栏主布局的中心地位。

职责：

- 渲染世界画卷背景。
- 渲染七大流域团块。
- 渲染鱼影。
- 处理流域点击、鱼种点击、键盘 focus。
- 将弹窗锚点位置传给 overlay 组件。

### `BasinNode`

单个流域团块。

输入：

- `habitat`
- `isSelected`
- `isDimmed`
- `fishCount`
- `onSelect`

输出：

- 可访问按钮。
- 样式变量：位置、主色、发光色、纹理强度。

### `FishShadowMarker`

单个鱼影。

输入：

- `fish`
- `isSelected`
- `isDimmed`
- `onSelect`

鱼影使用 CSS 或 SVG，不使用真实图片作为本阶段依赖。

### `MapInfoPopover`

统一弹窗组件，根据类型渲染流域或鱼种内容。

职责：

- 定位到流域或鱼影附近。
- 小屏幕时自动变为底部 sheet。
- 处理关闭、焦点回收和 Esc 键。

### `TuningDrawer`

复用现有 `FilterRail` 的筛选能力，但视觉上改成临时抽屉。

职责：

- 搜索。
- 筛选。
- 重置。
- 关闭后返回沉浸画卷。

## 数据流 / Data Flow

1. `App` 加载现有 `habitats` 和 `fish` 静态数据。
2. `filterFish` 根据筛选与搜索得到 `visibleFish`。
3. `WorldScrollCanvas` 根据 `habitats` 和 `visibleFish` 渲染流域与鱼影。
4. 用户点击流域，`activeOverlay` 变为流域弹窗，`selectedHabitatId` 同步更新。
5. 用户点击鱼影，`activeOverlay` 变为鱼种弹窗，`selectedFishId` 同步更新。
6. 用户打开调境抽屉并修改筛选，`visibleFish` 更新，画卷上的鱼影和流域状态同步变化。
7. 若筛选后当前选中鱼不再可见，则关闭鱼种弹窗或退回到对应流域弹窗。

## 响应式设计 / Responsive Design

桌面：

- 世界画卷占满视口。
- 弹窗定位在被点击元素附近。
- 调境抽屉从侧边浮出。

平板：

- 画卷仍完整显示，但流域标签默认更少。
- 弹窗宽度减少。

手机：

- 画卷采用可横向轻微拖动或缩放后的完整概览。
- 弹窗变成底部 sheet。
- 调境抽屉同样使用底部 sheet。
- 鱼影标签默认不展开，避免遮挡。

## 可访问性 / Accessibility

- 流域团块和鱼影必须是可聚焦按钮。
- `aria-label` 包含中文名和英文名。
- 弹窗打开后焦点进入弹窗，关闭后返回触发元素。
- Esc 关闭弹窗。
- `prefers-reduced-motion` 下减少循环流光和鱼影漂移。
- 弹窗文本对比度不低于现有暗色 UI 标准。

## 错误与空状态 / Error And Empty States

本应用仍为静态数据，无网络错误状态。

需要处理：

- 筛选无鱼种：世界画卷保留，流域变暗，弹窗提示“当前条件下暂无鱼种”。
- 当前选中鱼被筛掉：自动关闭鱼种弹窗，并保留当前流域。
- 弹窗锚点靠近视口边缘：自动翻转或退化为底部 sheet。
- 小屏幕文本过长：使用换行和最大宽度，不允许溢出。

## 测试计划 / Testing Plan

自动化测试：

- `filterFish` 既有测试继续保留。
- 新增或更新 `WorldScrollCanvas` 测试：七大流域按钮渲染、点击流域打开弹窗、点击鱼影打开鱼种弹窗。
- 新增 `MapInfoPopover` 测试：流域内容、鱼种内容、关闭行为。
- 新增 `TuningDrawer` 测试：筛选变更触发 `onFiltersChange`，重置可用。
- 更新 `App` 测试：默认首屏进入世界画卷，不再依赖三栏常驻详情。

手动/浏览器验证：

- 桌面首屏不空白，无框架错误 overlay。
- 七大流域位置符合相对地理预期。
- 流光循环可见但不影响阅读。
- 点击流域和鱼影能打开正确弹窗。
- 打开调境抽屉后筛选会改变画卷上的鱼影。
- 移动视口无文本重叠、溢出或遮挡关键点位。
- `prefers-reduced-motion` 下动画明显减少。

## 迁移策略 / Migration Strategy

1. 先保留现有数据、类型和筛选函数。
2. 新建 `WorldScrollCanvas`、`MapInfoPopover`、`TuningDrawer` 等组件。
3. 将 `App` 主布局替换为沉浸式 shell。
4. 逐步废弃常驻 `FilterRail`、`DetailPanel`、`FishList`、`CompatibilityBar` 在首屏的使用，但可复用其内容结构和测试逻辑。
5. 更新样式 token，避免变成单一深蓝或纯数据大屏。
6. 跑测试、构建和浏览器视觉 QA。

## 成功标准 / Success Criteria

- 默认首屏像一幅完整的水域世界画卷，而不是工具界面。
- 七大流域一眼能在世界相对位置上找到。
- 流光、水纹和鱼影让画面有循环流动感。
- 信息全部通过弹窗、sheet 或临时抽屉浮出，不占据首屏。
- 现有核心功能仍可用：选择流域、选择鱼、搜索、筛选、查看详情、查看混养摘要。
- 桌面和手机都无明显重叠、溢出或不可点击区域。
