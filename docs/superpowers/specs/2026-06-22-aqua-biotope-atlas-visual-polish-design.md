# Aqua Biotope Atlas 视觉精修一期设计文档

日期 / Date: 2026-06-22

## 阶段定位 / Phase Position

本阶段目标是把现有 **Aqua Biotope Atlas / 观赏鱼原生地探索图谱** 从“可用 MVP demo”推进到“可上线展示的互动艺术作品雏形”。

产品方向已调整为 **作品型上线 / artful showcase launch**，不是鱼友工具站，也不是完整鱼类资料库。它应该更像一幅可以探索、截图、分享的观赏鱼水域图谱：有知识骨架，但核心价值是视觉气质、游览感和记忆点。

视觉方向采用：

- **东方画卷感为主 / Scroll painting as primary direction**
- **自然博物图鉴感为辅 / Natural history plate as supporting language**

最终形态目标是横向长卷游览；本阶段先做过渡版本：保留现有三栏结构和核心交互，把中央信息图升级成“画卷式水域图谱”，把左右信息面板升级成“题签 + 博物图鉴卡”。

## 不变的产品边界 / Product Boundaries

本阶段不重新定义产品，不扩展成工具平台。

保留：

- 原生地 / biotope 作为主轴。
- 7 个原生地。
- 鱼种选择、搜索、筛选、详情、混养提示。
- 静态数据。
- 桌面优先，移动端可浏览。

不做：

- 账号、收藏、评论。
- 后端和 CMS。
- 电商、鱼缸推荐、饲养决策工具。
- 大规模数据扩展。
- 真正横向长卷架构重写。
- 海报导出和社交分享系统。

## 设计目标 / Design Goals

### 1. 从“暗色仪表盘”转为“水域画卷”

当前界面有完整功能，但气质偏 dashboard。第一期需要把视觉语言调整为低饱和、东方、水纹、绢本、墨彩鱼影的方向。

目标观感：

- 像一张展开中的水域图谱。
- 有宣纸、绢本、淡墨、水纹、题签的细节。
- 不做厚重古风，不做仿古装饰堆砌。
- 保持现代网页的可读性和交互清晰度。

### 2. 默认首屏可截图

用户不操作时，默认首屏也应该像一张完整作品截图。

首屏应满足：

- 标题明确：观赏鱼原生地探索图谱。
- 中央图谱是视觉焦点。
- 默认原生地和代表鱼能形成漂亮构图。
- 左右面板不抢戏，但质感到位。
- 页面不需要解释文字也能看懂主题。

### 3. 信息退后，氛围向前

数据仍然存在，但展示方式更“轻”。

- 参数不要像表格工具，应像图鉴标签。
- 混养信息保留，但不成为第一视觉重点。
- 鱼种卡片和标签更像标本签、题签、旁注。
- 筛选栏从“表单”转为“调水 / 调境”的感觉。

### 4. 为最终横向长卷铺路

本期不直接做完整横向长卷，但组件和视觉应向未来形态靠近。

当前中心图谱应开始具备“画卷段落”的感觉：

- 原生地节点不只是点，而是画面里的水域片段。
- 鱼影围绕水域游动。
- 各水域之间有水流、墨线、地貌肌理连接。

## 视觉语言 / Visual Language

### 色彩 / Color

基调从纯暗色水族馆转为“深水绢本”：

- 背景：深青黑、墨蓝、带少量宣纸纤维纹理。
- 高光：淡金、青绿、灰蓝、贝壳白。
- 鱼影：低饱和墨彩，不使用过强霓虹光。
- 警示色：弱化红橙，只用于不建议混养等轻提示。

避免：

- 赛博霓虹感过强。
- 大面积紫蓝渐变。
- 卡片玻璃拟态过重。
- 单一深蓝主题。

### 材质 / Texture

需要引入轻量材质层，但不依赖大图片。

可用方式：

- CSS radial / linear texture。
- 细噪点背景。
- 纸纤维感 overlay。
- 水纹线。
- 淡墨晕染。

材质应该低存在感，不能影响文字可读性。

### 字体与排版 / Typography

整体中文优先，保留英文术语作为图鉴气质。

- 主标题：中文更大，英文作为小字题签。
- 原生地：中文名 + 英文名并列。
- 鱼种详情：中文名主导，学名 italic。
- 参数：短标签 + 数值，像标本签。

不引入难以加载的在线字体。优先使用系统字体，通过字距、字号、大小写和颜色建立气质。

### 图形语言 / Graphic Motifs

可使用：

- 题签 / label strip。
- 印章式小标记，但必须克制。
- 水纹线 / current lines。
- 墨点 / wash stains。
- 鱼影剪影 / fish silhouettes。
- 原生地淡色团块 / habitat wash patches。

避免：

- 复杂装饰边框。
- 过度仿古卷轴两端。
- 大量图标堆叠。
- 与水域主题无关的传统元素。

## 核心界面设计 / Core UI Design

### App Shell

页面整体改为“画卷桌面”：

- 背景从暗色径向光改为深水绢本底。
- 内容区不再像 dashboard grid，而像铺在底纹上的一幅图谱。
- 三栏结构保留，但视觉上降低卡片感。
- 边框从玻璃线变成淡墨线 / 金线。

桌面布局：

- 左侧：生态条件题签 / tuning panel。
- 中央：水域画卷图谱 / scroll-map canvas。
- 右侧：鱼种博物图鉴签 / specimen plate。
- 底部：混养关系作为低调旁注。

移动布局：

- 单列。
- 顶部、筛选、图谱、鱼列表、详情依次排列。
- 不追求首屏完整展示全部内容，保证可读和无横向溢出。

### TopBar

从产品 header 变成画卷题头。

内容：

- Aqua Biotope Atlas 作为英文题签。
- 观赏鱼原生地探索图谱作为主标题。
- 当前原生地作为副题。
- 搜索框保持，但视觉上像题签输入，而不是普通表单。

设计要求：

- 首屏标题有作品感。
- 不做营销 hero，不加解释大段文案。
- 搜索控件不能压过标题。

### FilterRail

筛选栏改成“调境 / Ecology Tuning”。

视觉：

- 像卷边旁注或水质调境签。
- select 和 range 控件要更轻，不像后台表单。
- “重置筛选”可以像一个印章式按钮，但文字仍清晰。

交互不变：

- 原生地。
- 水温。
- pH。
- 性格。
- 最大体长。
- 色彩。
- 饲养难度。

### HabitatMap

这是本阶段最重要的改造对象。

当前节点地图升级为“水域画卷图谱”：

- 原生地节点改成水域墨彩团块。
- 每个 habitat 有更明显的地貌氛围：黑水、溪流、岩岸、稻田、珊瑚浅滩等。
- 鱼标记不再像地图 marker，而像轻微游动的鱼影。
- 鱼名默认不全部展开，避免文字拥挤；选中、hover、focus 时显示。
- 当前选中 habitat 应有更明显但克制的墨彩高亮。

中国南方溪流与稻田水域应保留明显位置和视觉身份，避免被东南亚/南亚节点吞掉。

### FishList

鱼种列表从结果条改成“鱼签横列”。

视觉：

- 单个鱼按钮像小题签 / specimen tag。
- 选中态像淡金描边或轻印章。
- 英文名作为小字，中文名为主。
- 保持横向滚动，适合未来长卷过渡。

### DetailPanel

详情面板改成“博物图鉴签”。

结构保留：

- 鱼影视觉区。
- 英文 common name。
- 中文名。
- 学名。
- 栖息地短句。
- story note。
- 参数 grid。
- visual tags。

视觉升级：

- 鱼影更像墨彩标本插图，而不是渐变胶囊。
- 参数格像小标本标签。
- story note 像旁注题跋。
- 学名保留 italic 图鉴感。

### CompatibilityBar

混养关系在艺术作品里不是主功能，需要降噪。

方向：

- 从底部大信息栏改成“关系旁注 / compatibility note strip”。
- 三类关系保留，但更轻。
- 文案继续保持谨慎，不输出绝对建议。
- 在移动端可自然堆叠到详情后。

## 内容与文案 / Content And Copy

本期不大改数据，但会调整可见文案语气。

文案原则：

- 少解释功能，多写意象。
- 保持科学名和参数准确。
- story note 更像图鉴旁白，而不是产品说明。
- 不使用“专业工具”“智能推荐”等工具型表达。

示例语气：

- “暗水中闪过蓝红细线。”
- “清澈溪流里的本土小型群游。”
- “硬碱湖岸的高对比色彩。”

不写：

- “点击这里开始探索。”
- “本工具可帮助你快速筛选。”
- “最适合混养。”

## 动效 / Motion

动效要轻，像水和鱼的呼吸。

包含：

- 鱼影轻微漂移。
- 水纹慢速移动。
- habitat hover 有墨色晕开感。
- 选中鱼时详情图鉴轻微浮现。

要求：

- 支持 `prefers-reduced-motion`。
- 不做大幅视差。
- 不影响阅读。
- 不依赖重型动画库。

## 技术设计 / Technical Design

本阶段以 CSS 和组件局部重构为主。

### 新增或调整的边界

新增：

- `src/data/visualTheme.ts`：集中管理 habitat visual style、fish color token、ink wash 变量。现有 `src/data/colorTokens.ts` 的鱼色 token 可迁入或被该文件复用，但对外应形成一个统一视觉 token 入口。
- `src/components/ScrollMapFrame.tsx`：承载画卷式背景、水纹、墨色团块，让 `HabitatMap` 专注于节点和鱼影交互。

调整：

- `HabitatMap.tsx`：拆分 habitat wash、fish silhouette、waterline layers。
- `DetailPanel.tsx`：抽出 fish artwork token 映射，减少内联样式散落。
- `styles.css`：按模块整理，避免继续堆成单一长文件；如果不引入 CSS modules，也至少用清晰分段注释。

不建议：

- 引入 Three.js。
- 引入大型 UI 组件库。
- 引入复杂动画库。
- 为一期接入图片资源流水线。

## 实施顺序 / Implementation Order

建议分 5 个小阶段：

1. **视觉 token 与全局底纹**
   - 建立色彩、材质、边框、题签变量。
   - 重做 app shell / top bar / panel 基调。

2. **中心图谱画卷化**
   - 重做 habitat node 视觉。
   - 加水纹和墨彩团块。
   - 优化鱼影和标签显隐。

3. **图鉴详情精修**
   - 重做鱼影视觉区。
   - 重做参数 grid 和 story note。
   - 保持详情信息完整。

4. **筛选与鱼签精修**
   - FilterRail 改成调境签。
   - FishList 改成鱼签横列。
   - 空状态保持克制。

5. **QA 与截图态**
   - 桌面 1280x720 / 1440x900。
   - 移动 390x844。
   - 默认首屏截图态。
   - 中国水域选中截图态。
   - 白云金丝详情截图态。

## 测试与验收 / Testing And Acceptance

### 自动验证

必须保持：

- `npm test` 通过。
- `npm run build` 通过。
- 数据完整性测试通过。
- 核心交互测试通过。

新增或调整测试：

- HabitatMap 颜色 token 不回退为无效业务色名。
- 空状态 reset 仍清空搜索和筛选。
- FishList 选中态和 aria pressed 保持。

### 浏览器 QA

必须检查：

- 首屏不是空白。
- 无 Vite / React error overlay。
- console 无相关 error / warning。
- 桌面无横向溢出。
- 移动无横向溢出。
- 原生地标签不互相遮挡。
- 鱼标签默认不造成文字拥挤。
- 选中鱼后详情和兼容旁注可读。
- `prefers-reduced-motion` 下无强制动画。

### 上线感验收

这不是功能 checklist，而是审美验收：

- 默认页截图是否像一个可分享作品。
- 中国南方水域截图是否有辨识度。
- 详情面板是否像图鉴，而不是后台详情卡。
- 页面是否从“查资料”转成“游览图谱”。

## 风险与取舍 / Risks And Tradeoffs

### 风险 1：过度古风

如果加入太多卷轴、印章、边框，容易廉价。解决方式：只借“材质、题签、水墨留白”，不堆传统元素。

### 风险 2：可读性下降

艺术化容易牺牲信息。解决方式：参数、学名、按钮状态保持清晰；背景纹理必须低透明。

### 风险 3：一次重构过大

直接做横向长卷会拖长周期。解决方式：本期只做过渡版，保留三栏和状态逻辑。

### 风险 4：视觉资产不足

没有真实鱼图或插画会影响精致度。解决方式：一期用 CSS 鱼影和墨彩剪影建立风格，后续再考虑生成插图资产。

## 一期完成定义 / Definition Of Done

本期完成后，项目应达到：

- 打开页面即有东方水域图谱的艺术气质。
- 中央图谱是视觉焦点。
- 默认首屏、中国水域、白云金丝详情三种状态都适合截图分享。
- 现有核心功能不退化。
- 桌面和移动端无明显布局错误。
- 测试和构建全部通过。
