# Aquaworld v0.11 — 内容审阅 & 视觉升级规划

## 背景

v0.1 已完成上线（woohey.github.io/Aquaworld），工程基线稳固：37 测试全绿、构建通过、资产优化至 ~770KB。

但 v0.1 有两个关键短板：

1. **内容未经审阅** — 7 个生境的生态框架、42 条鱼种的数据准确性、命名科学性，均未经过系统化的核查。部分栖息地边界模糊（如"浅水缓流与适应型水域"实际上是跨洲际的杂糅品类）。
2. **视觉未经过设计** — 世界卷轴背景图是通用世界地图，缺乏为图谱目的定制的视觉语言；每个生境没有专属的视觉辨识系统。

v0.11 的目标是解决这两个问题，将 Aquaworld 从"可运行的 demo"提升到"值得信任的图谱产品"。

---

## Part 1：内容审阅框架

### 1.1 七大流域独立审阅

每个生境分配一个子智能体进行专项研究，输出标准化的审阅报告。审阅维度：

| 维度 | 审阅要点 |
|---|---|
| **命名科学性** | 中文名、英文名、ID 是否准确反映该水域的生态特征？ |
| **生态描述完整性** | 水体类型、底质、光照、流速、植被、季节性变化是否描述充分？ |
| **水质参数准确性** | 温度、pH、硬度范围是否与权威资料一致？ |
| **代表鱼种合理性** | 每条鱼是否确实是该生境的典型物种？有没有更合适的替代？有没有不应该在此的鱼？ |
| **鱼种数据准确性** | 学名、体长、习性、饲养难度是否与鱼类学资料一致？ |
| **混养关系合理性** | 兼容性判断是否有依据？ |
| **叙事性** | 每条鱼的 storyNote 是否提供了图谱式洞察而非空洞描述？ |

审阅输出格式（每个生境一份）：

```markdown
# [生境名称] — 内容审阅报告

## 命名审阅
- 现有名称：xxx
- 建议：保留 / 修改为 xxx
- 理由：xxx

## 生态框架审阅
- 现有描述：[当前描述]
- 修改建议：[逐条修改]
- 补充内容：[季节变化、底质、植物群落、生态关系]

## 代表鱼种审阅
| 鱼种 | 判定 | 问题 | 建议 |
|---|---|---|---|
| Neon Tetra | 保留 | - | - |
| Cardinal Tetra | 保留 | - | - |
| Discus | 保留但需标注 | 野外捕捞 vs 人工繁殖差异大 | 补充养殖来源说明 |

## 单项数据审阅（每条鱼）
- 学名核查：
- 参数核查：
- 叙事性评级（A/B/C）：

## 鱼种调整建议
- 移除：xxx（理由）
- 新增：xxx（理由）
```

### 1.2 特别关注：栖息地 4 的结构问题

"浅水缓流与适应型水域"（south-asian-slow-waters）当前包含：
- Guppy（南美 Trinidad/Venezuela）
- Platy（中美洲 Mexico）
- Molly（中美洲至美国南部）
- Medaka Ricefish（东亚 Japan/China/Korea）
- Small Barb（南亚）
- Blue Gourami（南亚至东南亚）

这些鱼种来自 4 个不同的大陆，唯一的共同点是"适应力强、常见于社区缸"。

**建议方案**：将此栖息地重新定位为 **"水族贸易中的适应型鱼种"**（Aquarium Adaptables / 适应型观赏鱼群落），明确声明这不是一个地理栖息地，而是一个**文化/贸易节点**——反映水族贸易中那些已高度适应人工环境、跨越地理边界的鱼类。这个处理方式符合图谱的诚实性，也提供了一个有趣的叙事角度（人工选择与野生适应的对比）。

### 1.3 审阅优先级

| 优先级 | 内容 | 原因 |
|---|---|---|
| P0 | 栖息地 4 结构重组 | 当前是最大的内容缺陷 |
| P0 | 全部 42 条鱼的数据核查 | 图谱的权威性依赖于此 |
| P1 | 7 个栖息地的生态描述深化 | 影响用户体验 |
| P1 | 学名、中文名规范审查 | 专业度的基本要求 |
| P2 | 混养关系交叉核对 | 影响用户信任但不影响浏览体验 |
| P2 | 鱼种照片补充 | 15/42 覆盖率偏低但非阻塞 |

## Part 2：视觉升级方案

### 2.1 问题分析

当前世界卷轴的视觉层次：

```
背景层 → 纸纹理 → 鸟群 → 流光 → 波浪 → 内容（盆地节点 + 鱼影）
```

这个层次结构本身是好的，但背景图是一张通用世界地图，缺乏为 Aquaworld 定制的水域生态视觉。盆地节点虽然用了 `colorPalette` 驱动的 CSS 圆域，但与底层地图之间缺乏视觉一致性——节点是抽象的 CSS 图形，而背景是一张卫星地图。

### 2.2 方案：分两阶段

#### 阶段 A：低成本优化（本次 v0.11 执行）

1. **替换世界卷轴背景图** — 生成一张与 Aquaworld 调性匹配的风格化世界水域图（AI 生成，偏自然历史插画风格）。关键要求：
   - 不是卫星地图或政治地图
   - 表现水域分布（河流、湖泊、珊瑚礁区域）
   - 与卷轴的青绿色调一致
   - 7 个生境区域有微妙的色彩区分

2. **生境节点视觉强化** — 当前节点的 CSS 圆形区域已经不错，但可以增加：
   - 每个生境的微缩图标/符号
   - 选中状态下更丰富的动画

3. **生境浮层背景图** — 当前 `biotope-layer-background.webp` 是通用背景，替换为每个生境专属的水下场景插图（AI 生成，7 张）

#### 阶段 B：深度设计（v0.2 执行）

- 手绘风格的卷轴背景
- 每个生境的定制生物群落插图
- 鱼种卡片的正规视觉设计系统

### 2.3 阶段 A 资产清单

| 资产 | 数量 | 生成方式 | 规格 |
|---|---|---|---|
| 世界卷轴背景图 | 1 张 | AI 生成 (Gemini) | 1672×941 WebP |
| 生境专属水下背景 | 7 张 | AI 生成 (Gemini) | 1672×941 WebP |
| 生境微缩符号 | 7 个 | SVG/CSS 实现 | — |

## 已确认决策

| 决策项 | 选择 |
|---|---|
| 栖息地 4 | **重组**为「适应型观赏鱼群落」(Aquarium Adaptables)，诚实声明非地理栖息地 |
| 视觉风格 | **东方画卷** — 水墨/工笔/山水画的东方美学 |
| 鱼种数量 | **保持 42 条底线** — 每栖息地 6 条，质量优先，不合适则替换 |
| 执行方式 | **分批审阅** — 先试点栖息地 4（问题最突出），满意后铺开其余 6 个 |

## Part 3：实施计划

### 3.1 执行顺序（分批）

```text
第一轮（试点）：栖息地 4「适应型观赏鱼群落」审阅 + 修改
    ↓ 确认审阅质量和流程
第二轮：其余 6 个栖息地并行审阅（子智能体 × 6）
    ↓
内容修改（基于全部 7 份审阅报告）
    ↓
视觉资产生成（东方画卷风格，8 张 AI 图片）
    ↓
视觉集成
    ↓
验证上线
```

### 3.2 阶段分工

#### Phase 1：试点审阅 — 栖息地 4「适应型观赏鱼群落」

子智能体审阅当前 6 条鱼（guppy, platy, molly, medaka, small-barb, blue-gourami）：
- 查证学名、体长、原产地、水质参数准确性
- 评估"适应型观赏鱼群落"的命名与框架是否合理
- 判断这 6 条鱼是否适合作为"适应力"主题的代表
- 输出标准化审阅报告

试点通过后确认审阅模板质量，进入第二轮。

#### Phase 2：规模化审阅（子智能体 × 6，并行）

使用与 Phase 1 相同的标准化模板，6 个子智能体并行审阅：

| 子智能体 | 栖息地 | 范围 |
|---|---|---|
| A | 亚马逊黑水 | neon-tetra, cardinal-tetra, discus, apistogramma, pencilfish, corydoras |
| B | 东南亚溪流 | betta, harlequin-rasbora, pearl-gourami, tiger-barb, zebra-danio, rainbow-shark |
| C | 非洲裂谷湖 | mbuna-cichlid, shell-dweller, frontosa, electric-blue-hap, blue-dolphin-cichlid, butterfly-peacock-cichlid |
| D | 中美洲河流 | swordtail, firemouth-cichlid, livebearer, convict-cichlid, texas-cichlid, pearlscale-cichlid |
| E | 珊瑚礁浅滩 | clownfish, blue-tang, yellow-tang, damselfish, reef-goby, flame-angelfish |
| F | 中国南方溪流 | white-cloud-minnow, paradise-fish, roundtail-paradise-fish, chinese-bitterling, chinese-medaka, stream-goby |

#### Phase 3：内容修改（主智能体）

基于全部 7 份审阅报告，统一修改：
- `src/data/habitats.ts` — 栖息地数据修正
- `src/data/fish.ts` — 鱼种数据修正（替换不合适鱼种，保持 42 条总量）
- `src/data/fishMedia.ts` — 照片 URL 补充

#### Phase 4：视觉资产生成（AI 生成 × 8）

东方画卷风格（统一 prompt 模板确保风格一致）：
- 1 张世界卷轴背景图 — 水墨山水风格的写意水域地图
- 7 张生境专属水下背景 — 工笔水墨风格的水下场景插画
- 规格：1672×941 WebP

#### Phase 5：视觉集成（主智能体）

- 替换 `src/assets/` 中的世界卷轴背景图
- 修改 BiotopeLayerOverlay 按 habitat 切换专属背景图
- CSS 微调东方画卷风格与现有视觉层次协调

#### Phase 6：验证

- `npm test` 全绿
- `npm run build` 通过
- 逐生境浏览检查数据和视觉

## Part 4：风险与缓解

| 风险 | 缓解 |
|---|---|
| AI 图片风格不一致 | 统一东方画卷 prompt 模板 + 参考图锁定风格 |
| 审阅质量参差不齐 | Phase 1 试点先锁定模板，Phase 2 复用 |
| 栖息地 4 重组数据迁移 | 保持 ID 不变，只改 name/englishName/description |
| 学名/参数核实困难 | 子智能体通过 WebSearch 交叉验证多个权威来源 |
